## Project Overview
**PraveshPath** (formerly CED Platform) — A web app for Haryana university admissions eligibility matching. Students enter their academic profile and the platform auto-matches them with eligible programmes across Haryana universities based on qualification, marks, stream, and subjects.

- **Backend:** Django 6 + Django REST Framework, JWT auth (simplejwt + token blacklist), Google OAuth
- **Frontend:** React 19 + Vite 8, React Router, Axios, PWA enabled
- **Database:** SQLite (local), PostgreSQL (Railway production)
- **Local repos:** `/Users/aarya/CED_Platform/backend` and `/Users/aarya/CED_Platform/frontend`

- **Removed** the 4-step wizard with UG/PG card selection (Step 1: Select Level)

### 6. UI Overhaul (evening session)
- Complete redesign: CSS custom properties design system, SVG icons replacing emojis
- Frosted glass navbar with mobile hamburger menu + slide-out drawer
- Home page: gradient hero, animated stat counter (IntersectionObserver), features grid, how-it-works flow, CTA card, footer

- Profile wizard: SVG step indicators with glow ring, pill-shaped checkbox chips

- Results: pill-shaped badges, spinner loading, consistent `var(--radius-xl)` styling
- GJUS&T admission portal link updated across all 69 admission cycles
- PWA enabled in dev mode (`devOptions: { enabled: true }`), Apple meta tags added

### 7. Matching Engine Optimization (evening session)
- Eliminated double query in `find_eligible_courses` — was collecting IDs then re-querying, now returns list directly
- Added DB-level `course__level__in` filter (was filtering in Python after fetch)
- Changed `eligible.count()` → `len(eligible)` in views (avoids extra DB round-trip)
- Added `db_index=True` on `Course.level` and `Course.stream`
- Result: 13-40ms per match query (was significantly slower)

### 8. Database Optimization (evening session)
- Added indexes on `is_active` (University, Course, UniversityCourse), `academic_year`, `status` (AdmissionCycle)
- Fixed N+1 in `universities_with_programmes`: 1 query per university → 2 queries total via `Prefetch` with `to_attr`
- `UniversityCourseViewSet` list: added filtered prefetch for `current_cycles`, serializer uses prefetched data
- `university_programmes`: filtered `Prefetch` for 2026-2027 cycles only (was fetching all years)
- `platform_stats`: 4 separate COUNT queries → 3 via `aggregate`
- All major endpoints now run with constant query count regardless of data size

### 9. Security Hardening (evening session)
**Backend:**
- `SECRET_KEY` crashes at startup in production if env var missing (was insecure fallback)
- `DEBUG` defaults to `False` (was `True`)
- Default DRF permission changed to `IsAuthenticated` (was `AllowAny`)
- Rate limiting: 30/min anonymous, 120/min authenticated, 5/min on login/register/google-auth
- JWT token blacklisting enabled (`rest_framework_simplejwt.token_blacklist` + `BLACKLIST_AFTER_ROTATION`)
- Production security headers: HSTS (1 year), SSL redirect, secure cookies, referrer policy, `SECURE_PROXY_SSL_HEADER`
- Google OAuth now verifies `email_verified` claim (rejects unverified emails)
- `quick_match` rewritten: uses in-memory `StudentProfile` object (zero DB writes, no orphan rows)
- Percentage fields (10th, 12th, graduation) validated to 0-100 range via `MinValueValidator`/`MaxValueValidator`
- Removed hardcoded Google OAuth client ID fallback from settings

**Frontend:**
- Content Security Policy meta tag: restricts script/style/connect sources
- `upgrade-insecure-requests` meta tag for HTTPS enforcement
- API cache (`api-cache`) cleared on logout (PWA service worker was persisting authenticated responses)
- `.env` added to `.gitignore` (was missing, could be accidentally committed)

## Deployment Architecture

```
GitHub (Rajtaya/PraveshPath)
  │
  ├── Backend Service (railway up from backend/)
  │   ├── Nixpacks build (auto-detected Python)
  │   ├── railway.toml: migrate → collectstatic → gunicorn
  │   ├── PostgreSQL (Railway managed, internal network)
  │   └── WhiteNoise (static files)
  │
  └── Frontend Service (railway up from frontend/)
      ├── Nixpacks build (auto-detected Node/Vite)
      ├── PWA (service worker + manifest)
      └── Env: VITE_API_URL, VITE_GOOGLE_CLIENT_ID
```

**NOTE:** Deployment currently uses `railway up` (CLI upload), NOT GitHub auto-deploy. To enable auto-deploy:
1. Railway dashboard → each service → Settings → Source → Connect `Rajtaya/PraveshPath`
2. Backend root directory: `backend`
3. Frontend root directory: `frontend`

### Key Deployment Files
| File | Purpose |
|------|---------|
